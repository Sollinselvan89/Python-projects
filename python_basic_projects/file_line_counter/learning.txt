# ✅ **FILE LINE COUNTER — LEARNING SUMMARY**

## **1. File Handling Basics (What You Reinforced)**

* How to open files using `with open(filename, "r")`.
* That `with` automatically closes the file — no need for `file.close()`.
* Three main reading methods:

  * `read()` → whole text as a single string
  * `readlines()` → list of lines
  * Iterating the file → one line per loop (what you used)

---

## **2. Correct Way to Count Lines**

* You first tried counting characters (by looping over the string from `read()`).
* You learned that the **file object is iterable**, and each iteration gives you **one entire line**, not words, not characters.
* So this is correct:

  ```python
  for line in file:
      count += 1
  ```

## **3. Difference Between Line vs Word vs Character**

You asked:

> “What is a line? Does iteration count lines or words?”

Learnings:

* A **line** ends at a newline character (`\n`).
* Looping over a file counts **lines**, not words or characters.
* `read()` gives a string → character-level iteration.
* Iterating the file directly gives **line-level iteration**.
